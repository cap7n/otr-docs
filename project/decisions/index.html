    Decision Log¶

    Decisions that shape the game, newest first. When something gets decided in a design conversation, it gets a line here, with the why, so future-us doesn't relitigate it.

    Date Decision Why
    2026-07-14 Pratina - the harvestable residue life leaves in matter (a blend of prana + patina) is the substance the Prospector farms: loot value = Pratina density, and droid batteries, the car, and the structure all run on it. Gray-to-color is literal (a starving droid loses Pratina-perception). Fuel = refined Pratina - bought via the salvager resupply or scavenged from dense objects (closes the "where does fuel come from" question) Turns the abstract "value = arrangement" into a concrete substance that wires the economy, the fuel gauge, and the gray-screen intro to one root. See Pratina
    2026-07-14 The ripening model - Pratina must age and then spoils, so the structure is a cellar, not a conveyor: zone shuffling is stock rotation (hit each sample's harvest window), and the diegetic exit lights are that rotation schedule. Life also consumes Pratina, so enemies are crop management and the droids are pests/shrinkage The load-bearing rule: makes the zone-shuffle, the exit signals, and the enemy roster all legible as one farming operation that attentive players can read. See Pratina
    2026-07-14 Story spine: the megastructure is a Prospector that samples world-crust for arrangement (information), not material; the world is long-inhabited but unknowable (it samples across time, so knowledge never accumulates); the droids arrived via a breached waste chute (the crash = the diegetic junker start); personalities live in the droids (brains / speed-demon), expressed only through barks; the hub shop is a hacked salvager introduced as a mission; a peer radio hands out missions but never explains the place Gives level design an answer key for "what goes here" - every room reads as what the place was for and what interrupted it. Promotes the "sampling project" theory already noted in World & Structure to the designers' canon while keeping it unknowable to everyone inside. Full detail + proposals in World & Story
    2026-07-07 Music is diegetic - it plays from an in-car radio the crew switches on, not a score laid over the top (vibe leaning synthwave-rock a la Dance with the Dead; where/when it plays is TBD) Fits the diegetic-colour premise - plugged in = alive = the radio comes on - makes music something players control, and keeps the car the single source of power, colour, and sound. See Music & Soundtrack
    2026-07-07 Combat split - the driver deals no direct damage (ramming only); every gun belongs to the co-pilot, and the driver's kit is mobility + space-making (AoE stun / push-away). On foot, droids carry no firearm - tools and throwables only Makes the two seats inseparable in a fight: a driver alone can move and shove but cannot kill, so the co-pilot is structurally load-bearing, not merely "busy". Reconciling the built driver flamethrower/rocket-pot with the rule is open. See Weapons & Combat
    2026-07-06 Every cube has exactly 4 gates, one per wall - always physically present, open/closed is per-cycle state. Gates are shell infrastructure (the structure stamps them in), never zone content The shift system needs permanent sockets to toggle; players get a stable mental model (four walls, four gates, read the lights); one gate module built once and reused everywhere. Road-presence near a gate tells its history (worn connector = used before; no road = never seen open), lights tell its present
    2026-07-06 Art pipeline: no photo textures, materials are procedural Blender node graphs, baked to PBR texture maps (seamless tiles for big surfaces, per-object bakes for hero props) Photo textures clash with the stylized look; procedural keeps one visual language, and baking is the bridge to Godot (node graphs don't transfer) plus the mipmap fix below. First shipped: MegaPipeV2 + shell concrete v2. See Asset Pipeline
    2026-07-06 Giant-surface rule: high-frequency detail must be baked to textures; only low-frequency / per-instance variation (per-panel tint+flip, macro patches, floor grime) stays procedural in a slim shader Per-pixel shader math can't be mip-filtered, caused the shimmering-lines artifact on walls seen from ~2 km. Tiles can't vary per panel, shaders can; each layer does what only it can do. Reference implementation: mega_concrete_v2.gdshader
    2026-07-06 Blend prop materials are remapped by material NAME in the .blend.import, never node-path surface overrides in scenes Node-path overrides broke twice on the pipe when re-imports changed the scene tree; name-based remap survives re-UV, rename, and resize
    2026-07-06 ~~The shell's "digital squares" noise artifact is a deliberate look, do not fix~~ amended same day: shell moved to the baked hybrid and the clean look won, squares and drip streaks are zeroed in the tile author script (dials remain, can return per-zone) Original why: aligned-grid ghosting read as computed-not-poured, on-theme. After seeing it baked, it read as pixelation instead. The v1 shader guard comment only matters while v1 survives on reference surfaces
    2026-07-05 Engine upgrades are atomic: when one machine moves to a new Godot version, the other updates BEFORE opening the editor again. Corollary: never resolve git conflicts with "keep mine" on files you didn't intentionally edit A 4.6 editor opening the 4.7-migrated project silently stripped the new controller wiring from car.tscn and downgraded .import files, doors died and old sounds played on one machine, and the regression got pushed to main. Cost an evening of confusion; fixed in e911d5d
    2026-07-04 World: a megastructure lattice of ~2 km walled zones under a concrete sky; players are two droids powered by the car; garage = home; the lattice shifts between visits; expeditions can chain zones One fiction that explains the co-op pair, the seats, run variety, and authored chunks. Shifting = which level loads behind which gate, near-zero tech. See World & Structure
    2026-07-04 Zones are fully authored (no tile assembly) and anachronistic in theme, even time is uncertain Authored spaces have character; era-mixing means any art idea has a home, upgrades can span junkyard-to-futuristic, and the world's origin stays deliberately mysterious (retires the "who built it" question)
    2026-07-04 Battery leash: droids survive ~30-60s outside the car before gray-screen failure; the drone is limited by signal range, not battery Makes the car life itself (mirrors the intro), turns every on-foot excursion into a timed sortie and a two-player negotiation
    2026-07-04 Enemy direction: fewer interchangeable mobs, more one-of-a-kind bigger enemies that need a method to kill, not just bullets Landmark encounters fit authored zones; the weak-point machinery (locked gate) is the seed
    2026-07 OTR is 2-player co-op, period. No solo mode, no features that let one player run everything Every system is designed around two seats mattering; a solo fallback would quietly erode that. Removed the "Lone Wolf" modifier from the plans
    2026-07 This wiki is the single source of truth for design; old plan .md files archived Plan docs written before building never get updated after shipping, so they rot into lies (a feature audit found nitro/jump/wall-drive/flamethrower/rockets built while docs said "planned"). Rule: implementation status claims must be verified against code, never docs
    2026-07 Stay in Godot, do not port to Unreal Unreal has more raw power, but a 2-person team can't operate its complexity. Our scene (bounded static caverns + tunnels) is a Godot-friendly case: bake lighting instead of dynamic GI, fake scale with fog and light shafts. Porting = 6-12 month rewrite of working physics + netcode for no gameplay gain. See Setting & Vertical Scale
    2026-07 ~~Setting: an underground industrial civilization~~ superseded 2026-07-04 by the megastructure decision above The scale-as-horror rules survive unchanged on Setting & Vertical Scale; only the framing (underground rock) was replaced (walled zones + concrete sky)
    2026-07 Design wiki lives at cap7n.github.io/otr-docs, seeded from existing plan docs Shared overview for both devs; faster onboarding; records decisions
    2026-07 Tunnels confirmed as the level setting Confinement solves the blank-page problem; route = level in a driving game; plays to existing strengths (atmosphere stack, co-op seats). See The Tunnels
    2026-06 Distributed authority as the general netcode principle: whoever controls a thing simulates it and broadcasts its state (drone -> pilot's machine, on-foot player -> their own machine, car -> host). No anti-cheat validation beyond basic sender checks It's 2-player co-op with a friend, not competitive, responsiveness for the person holding the controls beats server authority. Decided early; recorded 2026-07-05 when a code audit almost "fixed" it as a security hole. RPC sender validation is consistency polish, not a requirement
    2026-04 Networking overhaul: single NetworkTickManager, StateBuffer + Hermite interpolation, no per-script sync RPCs Bandwidth control, consistent interpolation, one sync path to debug
    2026-04 Wall-driving via gravity shift, not sticky wheels Jolt wheels only push along local down; redirecting gravity makes them work unmodified
    - Jolt physics instead of Godot default Better vehicle behavior
    - Hand-placed enemy spawns, no procedural waves Pacing is authored, like encounter design in a session
    - Hazards must be readable, driver-focused, non-lethal individually No unfair surprise damage; danger comes from combinations

    (Older decisions reconstructed from project docs, add corrections if the why is off.)
